quikFilter demos

quikFilter is a suite of filtering functions written in jQuery that is released under open source licenses to allow anyone to use filtering on HTML DOM with relative ease. It contains quikFilter and multiFilter, both of which are demonstrated here. To view the source code, fork, post an issue or discuss this project, view it on Github.

quikFilter

quikFilter allows for very simple filtering with little work required. It filters based on custom attributes and is enabled with jQuery.

Usage

Each element (<li> in the case of the demo) has an attribute called "tag" which is then use in jQuery later when activating quikFilter. The HTML code of the list used in the demo is shown below to demonstrate the use of custom HTML attributes.
HTML Code

<strong>Shopping List</strong>:
<ul>
	<li tag="egg">Egg</li>
	<li tag="milk">Milk</li>
	<li tag="bread">Bread</li>
	<li tag="eggnog">Eggnog</li>
</ul>

quikFilter is enabled in the demo below with jQuery using an event handler:
Javascript/jQuery Code

$("#searchfield").on("keyup", {search:"tag"}, quikFilter);

You can use any event handler to perform the filtering but .on() gives the best response. The 'search' attribute is the name of the attribute for which the quikFilter will be using to filter. It allows you to use custom attributes (such as 'tag', used in this demo) to filter elements. This means that you can use attributes like 'name', 'address' or 'flavour' - anything you want.

multiFilter

Multifilter allows you to have multiple strings in the attributes which can then be filtered. All tag elements are entered into the html attributes, separated by commas.

Usage

multiFilter expects the attribute value to be separated by commas in a single string as shown in the HTML code of the demo, below.

<li tags="chocolate, cake">
	<a class="title">Simple Moist Chocolate Cake</a>
	<span class="badge">chocolate</span><span class="badge">cake</span>
</li>

multiFilter is enabled in the demo below with jQuery using an event handler:

$("#multisearchfield").on("keyup", {search:"tags"}, multiFilter);

You can use any event handler to perform the filtering but .on() gives the best response. The 'search' attribute is the name of the attribute for which the quikFilter will be using to filter.

Nested multiFilters

Nested filters allow grouping of filterable elements and hides empty 'nests'. It is easily enabled by adding a nested event.data parameter.

Usage

Nested multifilter does not strictly specify the generational gap between the nest and the element (that is, how many children deep) so as long as the element is a child of a child... of the nest, then it will be appropriately filtered. The code of the demo is shown below to demonstrate how this works with Bootstrap panels.

<div class="panel panel-default recipes">
	<div class="panel-heading">
	Entree
	</div>
	<div class="panel-body">
		<ul>
		<li food="dip, basil">
			<a class="title">Basil Dip</a>
			<span class="badge">dip</span><span class="badge">basil</span>
		</li>
		<li food="dip, guacamole">
			<a class="title">5 minute Guac</a>
			<span class="badge">dip</span><span class="badge">guacamole</span>
		</li>
		<li food="cheese, quick, paprika">
			<a class="title">Quick Paprika Cheeseball</a>
			<span class="badge">cheese</span><span class="badge">quick</span><span class="badge">paprika</span>
		</li>
		</ul>
	</div>
</div>

Nested multifilter is enabled in the demo below with jQuery using an event handler:

$("#nested_multisearchfield").on("keyup", {search:"food", nested:"recipes"}, multiFilter);

You can use any event handler to perform the filtering but .on() gives the best response. The 'search' parameter is the name of the attribute for which multiFilter will be using to filter. The 'nested' parameter is the class of DOM object which is the parent, grand-parent, grand-grand-parent (etc.) of the elements to be filtered. If there is no results within this object, the object will be hidden.
